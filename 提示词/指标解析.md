# 条件过滤树解析规则

## 任务目标
将用户输入的自然语言查询解析为嵌套的逻辑树结构，每个叶子节点代表一个原子过滤条件，包含字段名、运算符、值等信息，并按照预定义标签对字段进行分类。

## 核心规则
1. 将逻辑转换为标准中文运算符列表中的一项：`[等于,不等于,大于,小于,大于等于,小于等于,包含,在列表,区间,前匹配,后匹配,其他]`。
2. 构建嵌套树：使用 and / or 逻辑节点递归组合条件，精确表达用户的复合逻辑（如 (A and B) or C）。
	* 每个逻辑节点（and / or）的值是一个数组，数组元素可以是条件对象或嵌套的逻辑节点。
3. **禁止推断字段名**：除非用户明确提及。对于未提及字段名的条件，字段名应设置为空字符串，并将字段类型标记为“隐式”。

## 字段类型
1. 显式字段
*  判定标准: 用户在句子中明确的说出了字段名称
*  句式模式: `[字段名] [谓语] [值]` 或 `[字段名] [值]`
*  示例:
	* “创建日期为2026年” -> 字段名“创建日期”显式出现。
	* “客户等级是VIP” -> 字段名“客户等级”显式出现。

2. 隐式字段
*  判定标准: 用户未提及字段名，仅提供了值。
*  句式模式: `[谓语] [值]` 或 直接 `[值]` 修饰主体。
*  示例:
	* “VIP客户” → 用户没说“等级”，但“VIP”通常修饰等级 → field_type: 隐式
	* “2026年的订单” → 没说“日期”，但“2026年”修饰时间 → field_type: 隐式

## 标签分类
提取字段后，必须根据字段的语义或值将其归类到以下唯一标签列表中。若无法匹配，归类为“其他”。
*   **地理/位置** -> `地区` (如: 城市、省份、区域)
*   **内部架构** -> `部门`  (如: 销售部、市场部)
*   **内部架构** -> `组织` (如: 某子公司、分公司、某事业部)
*   **人员属性** -> `角色` (如: 经理、员工、负责人)
*   **时间颗粒度** -> `日期时间`
*   **属性/类型** -> `分类` (如: 行业、等级、颜色、状态)
*   **无法归类** -> `其他`

## 日期/时间值标准化 (关键规则)
当 `field_tag` 为 `日期/时间` 时，必须执行以下转换，**当前时间为** `{CurrentDate}`：
1.  **绝对化**: 将所有模糊时间词（Q1-Q4, 上周, 去年, 8月）转换为具体的 **YYYY-MM-DD** 格式。
2.  **区间化**:
    * 如果用户提到的是一个**时间段**（如'2026年', 'Q3', '8月'），则运算符必须设为 `区间`。
    * `field_value` 必须包含该时间段的 **[起始日期, 结束日期]**。
    * 若用户提到的是具体日期点（如“2026-03-05”），则运算符按实际逻辑处理，field_value 为单个日期。
3.  **计算逻辑 (基于当前时间)**:
    *   **年份**: '2026年' -> `[2026-01-01, 2026-12-31]`
    *   **季度 (Q1-Q4)**:
        *   Q1 → [2026-01-01, 2026-03-31]
        *   Q2 → [2026-04-01, 2026-06-30]
        *   Q3 → [2026-07-01, 2026-09-30]
        *   Q4 → [2026-10-01, 2026-12-31]
    *   **月份**: '8月' (假设当年) -> `[2026-08-01, 2026-08-31]`
    *   **相对词**:
        * “上周” → 计算上周一到上周日：[2026-02-23, 2026-03-01]
        * “去年” → 上一年全年：[2025-01-01, 2025-12-31]
        * “上个月” → [2026-02-01, 2026-02-28]
        * “最近7天” → 从今天往前推7天：[2026-02-27, 2026-03-05]

# 指标提取规则
1. **原子指标指标识别**：
   - 若指标是可直接聚合、与常量计算、与度量字段计算，则视为原子指标
   - 原子指标结构：
     - name: 指标名称
     - metric_type: 值 “原子指标”
     - base_metric: 从  `base_metrics` 选取的度量字段
     - formula: 此指标的计算公式，计算对象使用 `{base_metric}` 替换
     - column_name: 度量结果别名
     - dependent_dims: 计算此指标需要依赖的维度
     - filters: `条件过滤树` 的结构

2. **复合指标识别与拆分**：
   - 若指标包含 周期比较、涉及多个原子指标计算、非直接聚合运算，则视为复合指标。
   - 复合指标的作用: 将原子指标视作一个CTE子句，标识此指标的计算公式。
   - 对于复合指标应进行指标下钻，需明确其**计算公式**，并拆分为**多个原子指标子节点**，将子指标放入 children 继续拆分，直到所有子指标都是原子指标。
   - 计算公式中使用 `{children.column_name}`将计算对象替换。
   - 复合指标结构：
      - name:指标名称
      - metric_type:值“复合指标”
      - formula:此指标的计算公式，计算对象使用 `{children.name}` 替换
      - children:子指标信息列表，每个子指标结构同原子指标
 
3. **维度关联**：
   - 根据用户提问的业务背景和分析需求，结合 `上下文信息` 中的 `Dimensions`，判断复原子指标是否依赖于特定维度的计算，识别并填入 `dependent_dims`。
   
4. **指标条件识别**：**仅仅提取以下两种**
   - 指标全局条件：向自己提问：“此条件是否是计算完指标后对指标过滤？”若是则提取到 `having_filters`
   - 指标局部条件：向自己提问：“此条件是否作用于指标内部过滤？”若是则提取到 `原子指标.filters`
   - 提取规则见  `条件过滤树解析规则`

5. 涉及计算时优先考虑 `系统内置计算公式` 中定义的公式
6. 提取指标 **根名词** 填写到 `base_metrics`

请将提取到的指标归类为以下两类之一：
### 1. 基础指标
- **定义**：对单个可分析的数值字段进行直接聚合运算（求和、平均、计数、最大、最小等）得到的指标。
- **特征**：仅涉及**一个**可分析的数值字段，运算逻辑简单（SUM、AVG、COUNT、MAX、MIN）。
- **分类标签**：
  - **度量字段**：指可以被聚合计算的数值型字段（如销售额、利润、时长、年龄、收入等）
  - **实体**：指当用户明确需要计数时，背后的业务主体（如"用户数"背后的用户主体，但**不提取具体的用户ID值**）
- **示例**：
  - "总销售额" → 指标名称：销售额，tag：度量字段
  - "用户数" → 指标名称：用户数，tag：实体（注意：这里提取"用户数"而非"用户ID"）
  - "平均年龄" → 指标名称：年龄，tag：度量字段
  - "订单量" → 指标名称：订单量，tag：实体

### 2. 复合/衍生指标
- **定义**：通过以下方式之一得到的指标：
  a. **跨字段运算**：涉及两个或以上不同的度量字段（如比率、差值、加权）
  b. **时间周期对比**：同比、环比、定基比、YTD累计等
  c. **复杂逻辑**：包含条件判断、过滤、排名、百分位等
- **特征**：运算逻辑复杂，或涉及多个字段，或包含时间偏移。
- **提取原则**：**优先提取被计算的核心对象**，即参与运算的基础度量字段，而非计算方式本身。
- **分类标签**：
  - **度量字段**：当有明确的被计算对象时（如"XX的同比"），提取该对象，tag设为"度量字段"
  - **其他**：当无明确的被计算对象，指标本身已是复合指标名称时（如"毛利率""客单价"），tag设为"其他"
- **示例**：
  - "合同金额的同比" → 指标名称：合同金额，tag：度量字段
  - "利润的环比增长率" → 指标名称：利润，tag：度量字段
  - "销售额同比增长率" → 指标名称：销售额，tag：度量字段
  - "客单价" → 指标名称：客单价，tag：其他
  - "毛利率" → 指标名称：毛利率，tag：其他
  - "留存率" → 指标名称：留存率，tag：其他

## 提取规则
1. **过滤非指标字段**：首先根据排除规则，过滤掉所有不应作为指标提取的字段。
2. **识别所有指标**：从过滤后的用户输入中找出所有需要计算的指标项。
3. **判断指标类型**：根据上述定义，判断每个指标是"基础指标"还是"复合/衍生指标"。
4. **确定指标名称和tag**：
   - 对于**基础指标**：
     - 输出该指标对应的**指标名称**（如果是计数类，输出"用户数""订单量"等计数概念名称，而非背后的ID字段）
     - 根据基础指标tag判断规则设置`tag`值
   - 对于**复合/衍生指标**：
     - 如果指标描述中包含**被计算的核心对象**（如"销售额的同比"），优先提取该核心对象名称，`tag`设为"度量字段"
     - 如果指标本身已是复合指标名称且无明确核心对象（如"客单价""毛利率"），保留该名称，`tag`设为"其他"
5. **去重处理**：相同的指标名称只输出一次。
