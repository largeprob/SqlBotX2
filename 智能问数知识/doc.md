1. 什么是粒度？
粒度是一行数据所代表的业务含义的详细程度。
它描述了“当我查看数据时，每一行代表什么”。
在原始事实表中，粒度由表的每一行记录决定（如一行代表一笔订单、一次点击、一个客户）。
在聚合查询结果中，粒度由 GROUP BY 的维度字段决定。

2. 粒度与可度量字段的关系
可度量字段（如销售额、数量）是被聚合的对象，而粒度是聚合的分组依据。
它们的关系是：

度量值 + 分组维度 = 带粒度的聚合结果

4. 但在数据建模中，我们可以说“表的粒度”
在数据仓库建模时，我们经常说“这张表的粒度是交易行级别”或“这张表的粒度是日汇总级别”。
这指的是该表每行数据代表的业务粒度，而不是某个具体字段的粒度。
事实表 fact_orders：粒度是每笔订单（一行一笔订单）

汇总表 agg_daily_sales：粒度是每天 + 每个产品（一行一天一个产品的汇总）
